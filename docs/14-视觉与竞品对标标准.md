# 14. 视觉与竞品对标标准

## 目标

PsyPic 最终视觉目标不是复刻 Liblib，而是在 GPT Image 专用商业创作场景里做到：

- 更快进入创作。
- 更少参数噪音。
- 更强商业模板感。
- 更清晰的结果复用路径。
- 更精致的灵感社区和作品详情。

一句话标准：

> 第一眼像专业商业创作工具，第二眼像高质量灵感站，真正使用时比大而全平台更快。

## 对标对象

主要参考：

- Liblib inspiration：灵感流、作品卡片、同款生成入口。
- Midjourney web：作品浏览、详情、参数复用。
- Canva：商业模板、尺寸选择、品牌资产。
- Figma / Linear：工具界面的密度、克制和效率。

不复制：

- 不做模型广场优先。
- 不把首页做成泛社区瀑布流。
- 不用大面积营销 hero 替代创作台。
- 不让用户先选复杂工作流才能生成图。

## 信息架构

第一层导航：

- 创作台。
- 商业模板。
- 灵感社区。
- 历史/素材。
- 设置。

创作台是默认首屏，不做纯营销首页。

## 创作台视觉标准

布局：

- 桌面端三栏：左参数、中结果、右历史/详情。
- 移动端：顶部结果区，底部抽屉式参数面板。
- 参数栏保持工作台密度，不做大卡片堆叠。

视觉：

- 背景中性、干净、低干扰。
- 参数区紧凑但可读。
- 生成按钮必须明显，但不喧宾夺主。
- 结果图必须是视觉焦点。
- 避免紫蓝渐变、装饰光斑、大面积营销风背景。

状态：

- 空状态展示商业模板入口和最近灵感。
- 生成中展示进度、耗时、可取消状态。
- 失败状态展示可理解原因和可执行修复。
- 成功状态突出下载、继续编辑、同款、发布。

## 商业模板视觉标准

模板中心按商业用途组织：

- 电商主图。
- 产品换背景。
- 广告 Banner。
- 社媒封面。
- 品牌海报。
- 人像头像。
- 应用首屏视觉。

模板卡片必须展示：

- 示例图。
- 使用场景。
- 推荐尺寸。
- 生成入口。
- 是否需要参考图。

模板表单：

- 使用结构化字段，不暴露长 prompt。
- 高级参数折叠。
- 每个模板有默认尺寸和质量。
- 支持品牌色、留白、构图、光线、文字区域等商业字段。

## 灵感社区视觉标准

信息流：

- 以图为主，但卡片信息必须服务商业复用。
- 默认按商业场景筛选，而不是只按风格筛选。
- 每张卡片有明确的“同款”入口。
- 卡片 hover 或移动端操作区显示收藏、同款、详情。

作品详情：

- 左侧大图或主视觉。
- 右侧显示场景、标签、作者、参数摘要、模板来源。
- “同款生成”是主按钮。
- “作为参考图继续编辑”是次按钮。
- prompt、参数、参考图公开状态必须清楚显示。

社区不能只好看，必须让用户能从灵感回到创作台。

## 组件标准

按钮：

- 工具按钮优先图标 + tooltip。
- 明确命令才使用文字按钮。
- 主操作按钮全局保持一致。

卡片：

- 只用于作品、模板、结果和弹窗。
- 不把页面 section 做成浮动卡片。
- 圆角 8px 以内，除非设计系统另有统一规则。

参数控件：

- 模型和模式用 segmented control。
- 尺寸用预设按钮 + 自定义输入。
- 数量用 stepper。
- 质量用 segmented control。
- 压缩率用 slider。
- 开关用 switch。
- 高级参数折叠。

图片结果卡：

- 固定宽高比，避免加载后布局跳动。
- 展示生成状态、尺寸、格式、耗时。
- 操作：下载、继续编辑、作为参考图、发布、收藏。

## 色彩和字体

方向：

- 主色可用于按钮和焦点状态，但整体不被单一色系统治。
- 商业创作台偏中性、清爽、专业。
- 灵感社区可更视觉化，但不牺牲阅读和操作。

避免：

- 大面积紫蓝渐变。
- 深蓝/暗黑单一风格。
- 米色/棕色单一风格。
- 纯装饰光球、光斑、抽象渐变背景。
- 文字压在复杂图片上导致不可读。

字体：

- 中文使用系统无衬线。
- 英文和数字使用系统无衬线。
- 不使用随视口宽度缩放的字体。
- 按钮和卡片文字必须保证移动端不溢出。

## 移动端标准

必须支持：

- 创建新图。
- 上传参考图。
- 查看结果。
- 下载。
- 继续编辑。
- 浏览灵感社区。
- 从社区一键同款。

移动端不强求高级批量管理，但主流程不能断。

## 性能标准

创作台：

- 首屏不被社区瀑布流拖慢。
- 图片懒加载。
- 大图使用缩略图和按需加载。
- 上传前压缩预览图，但原图按需要上传。

社区：

- 无限滚动或分页均可。
- 图片使用固定尺寸容器。
- 骨架屏不改变布局。
- 详情页优先加载主图和同款按钮。

## 验收标准

- 新用户进入首屏 10 秒内知道从哪里开始生成图。
- 商业模板入口在首屏可见。
- 生成结果的下载、继续编辑、发布路径清楚。
- 社区作品详情能一键回到创作台。
- 移动端没有明显文字溢出、按钮挤压和图片布局跳动。
- 视觉上不像通用后台，也不像营销落地页，而是专业商业创作工具。

## 暗色模式标准（Phase 6 落地后）

- 全站可在 AppTopNav 工具区切换三态：亮色 / 暗色 / 跟随系统。
- 暗色态 token 由 `app/globals.css` 的 `.dark { ... }` 集中定义，所有
  surface / text / border 颜色都通过 CSS var 读取，避免组件内硬编码 hex。
- 主题持久化至 localStorage（key `psypic-theme`），SSR 安全：服务端渲染
  统一为亮色，客户端 mount 后 `<ThemeNoFlashScript />` inline 注入同步
  脚本，避免暗色用户首次进入白屏闪烁。
- 用户系统偏好变化（OS 切夜间模式）实时跟随，无需重载页面。
- 暗色态下图片覆盖层 chip / 阅读卡片底色保持足够对比度，不出现纯白
  badge 浮在暗色页面上的视觉割裂。

## 移动端标准（Plan Task 6 落地后）

- 工作台移动端：左侧项目/对话栏 + 右侧 Inspector 都收纳进 shadcn Sheet
  抽屉，由 ChatHeader 汉堡按钮和 mobile-bottom-bar「打开参数面板」触发；
  Composer 留在 chat-workspace 底部，主 prompt 输入不被抽屉打断。
- 社区列表移动端：sort / scene / tag 三组 chip 行单行横滚，不再 wrap 多
  行把首屏作品压到 fold 之下。
- 作品详情 / 素材详情移动端：主图最大高度 55vh（桌面 calc(100vh-220px)），
  让「生成同款草稿」/「继续编辑」CTA 在首屏可达，不沉到 fold 之下。
- 主导航栏在移动端折叠为水平横滚条，工作台 / 灵感社区 / 设置 / 主题切换
  保持都能点到。

## ProjectSidebar Codex 风标准（plan slug `cosmic-tumbling-narwhal`）

- 项目不再 hardcoded：4 个内置项目（商业 / 社媒 / 广告 / 社区同款）作为 IndexedDB
  `psypic_projects` store 的 default seed，用户可新建（`+ 新建项目`）/ 重命名
  / 删除；最后 1 个项目时 AlertDialog 禁用 Action 防止把 sidebar 删空。
- 项目级 kebab DropdownMenu 6 项：固定项目 / 在资源管理器中打开 / 创建永久工作树 /
  归档对话 4 项 placeholder，重命名 + 移除 实做。
- 项目内 session 列表按时间分桶（**今天 / 昨天 / 本周 / 更早**，周一为周首），
  每桶 SectionHeading + count + 桶内按 latestNode.createdAt 倒序；空 sessions
  时显示「还没有生成记录」引导文案。
- 每个 session row 同时挂 ContextMenu（桌面右键）+ DropdownMenu（hover / 移动端
  三点按钮）11 项菜单，内容相同；复制会话 ID + 复制深度链接 实做（写入
  navigator.clipboard），其余 9 项走 SidebarToast「即将上线」/「桌面端功能」分诊。
- 整 sidebar 包在 `<SidebarToastProvider>` 里，所有占位动作统一通过极简 inline
  toast 反馈（不引 sonner / react-hot-toast，遵守 CLAUDE.md「不引入新状态库」
  边界）。
- 视觉细节：active project 进入 ProjectHeader（icon + 标题 + kebab）置顶；其他
  项目折叠为 ProjectSwitcher row 列表；session row hover 高亮 + active 走 accent
  描边；与桌面 Codex / ChatGPT sidebar 对标无明显割裂。
