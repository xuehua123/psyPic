# PsyPic 开发前方案文档

本文档集用于在正式开发前统一产品边界、API 参数、架构、安全、Sub2API 集成、开发阶段和最终封板标准。

## 当前结论

PsyPic 建议做成独立 AI 生图站，而不是嵌入 Sub2API 管理后台。Sub2API 负责账号、密钥、网关、额度、计费和风控；PsyPic 负责创作台、图片生成/编辑、素材库、历史、模板、批量任务和分享体验。

产品方向是商业创作台 + 灵感社区：先赢商业创作效率，再用作品发布、精选、收藏和同款生成补齐社区增长闭环。

默认技术路线：

- 除非另行指定，PsyPic 默认使用 Next.js App Router + TypeScript + BFF Route Handlers。
- MVP 使用 OpenAI Images API：`/v1/images/generations` 和 `/v1/images/edits`。
- 默认图片模型：`gpt-image-2`。
- 后续多轮编辑与上下文创作接入 Responses API 的 `image_generation` tool。
- 一键导入 Sub2API 密钥使用短期一次性 ticket，禁止使用 `?apiKey=...` 这种明文 URL 方案作为生产默认方案。
- PsyPic 后端/BFF 代持密钥更安全，前端只保存登录/session 状态。

## 推荐开发节奏

当前 V1.0 代码封板已完成，下一步是在 staging/production 环境做真实 Sub2API、对象存储、migration、压测和回滚演练：

1. [x] `v0.1`：项目骨架、手动配置、文生图基础。
2. [x] `v0.2`：Sub2API 一键安全导入、BFF 密钥代持。
3. [x] `v0.3`：文生图 alpha、完整参数、基础历史。
4. [x] `v0.4`：图生图、单参考图、继续编辑、公开试用验收。
5. [x] `v0.5-v0.9`：任务系统、遮罩、素材库、商业模板、管理端和社区基础。
6. [x] `v1.0`：社区互动、批量工作流、多参考图、E2E、移动端 smoke 和生产封板文档。

后续非 V1 阻塞项：团队空间、真实账单自动对账、Redis/DB-backed queue 横向扩展和更完整的生产观测。
