# 07. 验收清单

## MVP 验收

对应 `v0.4` 公开试用版本。

- [x] 独立站可部署。
- [x] 手动配置 Sub2API Base URL 和 API Key 后可以生图。
- [x] Sub2API 一键导入安全契约可用，导入响应不返回明文 Key。
- [x] 密钥不出现在 URL。
- [x] 文生图可用。
- [x] 图生图可用。
- [x] 支持 `gpt-image-2` 默认模型。
- [x] 支持尺寸预设和自定义尺寸校验。
- [x] 支持商业尺寸预设，单次只选择一个目标尺寸生成/下载。
- [x] 支持 `quality`。
- [x] 支持 `output_format`。
- [x] 支持 JPEG/WebP `output_compression`。
- [x] 禁止 `background=transparent`。
- [x] 不传 `input_fidelity`。
- [x] 支持结果下载。
- [x] 支持失败重试。
- [x] 支持基础历史。
- [x] 显示 request id、耗时、usage。
- [x] 超时、429、401、403、400 错误有明确提示。

## 测试场景

### 自动化回归状态

- [x] `pnpm e2e` 覆盖设置页手动 key、文生图、结果转参考图、遮罩图生图、流式 partial preview、并发 429、批量 Prompt、素材库同步、社区发布、社区排序筛选、点赞、收藏、同款隐私、举报、管理端 runtime settings / 举报处理 / 精选和移动端 smoke。
- [x] CI 执行 Playwright Chromium E2E。
- [ ] 真实 Sub2API launch ticket、生图额度、生产对象存储、压测和回滚演练仍需在 staging/production 环境验收。

### 正常场景

- [x] 生成 1 张 `1024x1024` PNG。
- [x] 生成 4 张 `1024x1024` JPEG。
- [x] 生成 `1536x1024` high。
- [ ] 生成 `3840x2160` high，若管理员和上游额度允许。
- [x] 上传 1 张参考图生成。
- [x] 上传多张参考图生成。
- [x] 使用 mask 局部编辑。
- [x] 流式生成并收到 partial image。

### 参数错误

- [x] 自定义尺寸不是 16 倍数时自动规整。
- [x] 超过最大边时报错或规整。
- [x] 长宽比超过 3:1 时拦截。
- [x] `output_compression` 在 PNG 下隐藏或不传。
- [x] `background=transparent` 被拦截。
- [x] `input_fidelity` 不被传递。

### 权限和额度

- [x] ticket 过期。
- [x] ticket 重复消费。
- [x] key inactive。
- [x] key quota exhausted。
- [x] key 不属于当前用户。
- [x] group 不是 OpenAI。
- [x] 同一用户已有 active 图片任务时返回 429。

### 稳定性

- [x] 上游 429。
- [x] 上游 500。
- [x] 生成超时。
- [x] 用户取消任务。
- [x] 上传大文件。
- [x] 网络中断或失败后可通过失败/取消任务重试恢复。
