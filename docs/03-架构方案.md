# 03. 架构方案

## 总体架构

PsyPic 采用独立站架构：

```text
Browser
  -> PsyPic Frontend
  -> PsyPic BFF/API
  -> Sub2API Gateway
  -> OpenAI / upstream account pool
```

职责划分：

- PsyPic Frontend：Project Workspace、Panel Shell、参数面板、上传、预览、版本流、Board Mode、历史、商业模板、素材库、灵感社区、交互。
- PsyPic BFF/API：session、密钥代持、图片代理、任务队列、对象存储、成本记录、项目/版本服务端化、作品发布和社区审核。
- Sub2API：用户密钥、分组、额度、计费、模型网关、上游调度。
- Object Storage：生成图、参考图、mask、Board export、导出 ZIP。
- DB：用户配置、任务、图片索引、模板、分享链接、公开作品、收藏、点赞、举报。

## 图片工作台分层

PsyPic 的长期架构以工作台为主壳：

```text
Workbench Shell
  -> Project Sidebar
  -> Main Canvas / Transcript
  -> Inspector
  -> Task Dock

Domain Services
  -> ImageService
  -> VersionGraphService
  -> AssetLibraryService
  -> JobRuntimeService
  -> TemplateService
  -> CommunityService
```

关键边界：

- Workbench Shell 只编排 UI，不直接调用 Sub2API。
- Version graph 第一阶段可 local-first，正式版迁移到 DB。
- JobRuntimeService 统一处理生成、流式、批量、导出、审核等任务状态。
- AssetLibraryService 统一处理图片资产、标签、收藏、相册、下载和访问权限。
- Provider boundary 保留 Sub2API / Responses image tool / 未来 OCR、去背景、超分等扩展点。

## 前端模块

### Workbench Shell

项目级创作壳：

- 左侧 Project Sidebar：项目、会话、版本入口。
- 中间 Main Canvas：画廊、对话式版本流、Board Mode、批量结果。
- 右侧 Inspector：参数、模板、参考图、素材、任务详情。
- 底部 Task Dock：运行中任务、批量队列、导出状态。

### Creator Workspace

核心创作界面：

- Prompt 输入。
- 参数面板。
- 参考图/遮罩入口。
- 生成按钮。
- 结果网格。
- 任务状态。

### Version Flow

版本流模块：

- 每次生成或编辑创建 version node。
- 支持恢复参数、回到版本、从此分叉。
- 分叉必须 non-destructive，不删除旧结果。
- 节点保存 prompt、参数、参考图、mask、usage、request id 和耗时。
- 如果来源是 Board Mode 画布，节点额外保存 board document id、board snapshot 和 board export asset id。

### Image Editor

图片编辑模块：

- 参考图上传。
- mask 画布。
- Board Mode 导出的 reference PNG / alpha mask。
- 结果转参考图。
- 局部替换。

### Board Mode

Board Mode 模块：

- 前端使用 BoardDocument / BoardLayer JSON 作为长期模型。
- 画布交互可以由 Konva + react-konva 承载，但第三方 canvas 实例不进入全局状态或服务端 API。
- 本地第一阶段保存到 IndexedDB；服务端化后进入 workbench boards API。
- 导出结果进入 ImageAsset / BoardExportAsset，再作为 `/api/images/edits` 的 `image` 或 `mask`。
- Board snapshot 与 version node 关联，支持恢复、分叉和搜索。

### History Gallery

历史模块：

- 任务列表和 version node 入口。
- 收藏。
- 搜索。
- 标签。
- 批量操作。

### Global Search / Command Surface

工作台搜索和命令入口：

- 搜索 project、session、version node、board、asset、prompt、tag、template、request id。
- 快速执行新建项目、打开设置、批量导出、从当前图分叉、发布作品等命令。

### Template Center

模板模块：

- 商业用途模板。
- 风格预设。
- Prompt 片段。
- 管理员推荐模板。

### Inspiration Community

灵感社区模块：

- 商业场景信息流。
- 作品详情。
- 一键同款生成。
- 点赞、收藏、精选、举报。
- 发布可见性和 prompt/参数/参考图公开控制。

### Settings

设置模块：

- API 配置。
- 导入状态。
- 默认参数。
- 存储策略。

## 后端模块

### Auth / Session

- 处理 Sub2API ticket 导入后的登录态。
- 前端不直接持有长期密钥。
- 支持手动 key 的安全保存策略。

### Sub2API Import

- 接收一次性 ticket。
- 向 Sub2API 验证并换取绑定信息。
- 创建 PsyPic session。

### Image Proxy

- 对外提供 PsyPic 内部接口：
  - `POST /api/images/generations`
  - `POST /api/images/edits`
  - `POST /api/responses/image`
- 后端带 Sub2API key 调用 Sub2API 网关。
- 对响应进行 usage、request id、错误标准化。

### Task Queue

- 记录任务生命周期，作为 Job Runtime 的第一阶段实现。
- 支持取消、重试、并发限制。
- 流式任务通过 SSE 转发 partial image。
- 后续批量生成、ZIP 导出、素材同步和社区审核任务复用同一状态模型。

### Storage

MVP 可使用 IndexedDB 保存历史，但正式站建议服务端存储：

- 数据库存任务元信息。
- 数据库存 project、session、version node 和 asset 索引。
- 对象存储保存图片文件。
- 图片用 hash 去重。
- 私有图片默认不可公开访问。

### Billing Snapshot

- 保存请求参数。
- 保存响应 usage。
- 保存估算成本。
- 保存 Sub2API 实际扣费结果。

### Community / Publishing

- 保存公开作品、标签、场景、模板来源。
- 保存点赞、收藏、举报、精选状态。
- 控制作品可见性：私有、链接可见、公开社区。
- 控制 prompt、参数、参考图和原始素材是否公开。
- 支持管理员下架和关闭公开发布。

## 数据流

### 工作台生成任务

```text
User submits from Project Workspace
  -> Frontend creates or selects project/session/version context
  -> PsyPic BFF validates params and ownership
  -> JobRuntime creates task
  -> ImageService calls Sub2API / upstream provider
  -> AssetLibrary persists result assets
  -> VersionGraph records node snapshot
  -> Frontend updates canvas, inspector and task dock
```

### 文生图

```text
Frontend submits prompt + params
  -> PsyPic BFF validates params
  -> PsyPic BFF calls Sub2API /v1/images/generations
  -> Sub2API forwards to upstream
  -> PsyPic receives image + usage
  -> Store image and task
  -> Frontend renders result
```

### 图生图/遮罩

```text
Frontend uploads image/mask
  -> PsyPic BFF validates size/type
  -> Upload to temporary storage or multipart memory
  -> Call Sub2API /v1/images/edits
  -> Store result and source references
  -> Render result and allow continue editing
```

### Board Mode 导出编辑

```text
User arranges assets in Board Mode
  -> Frontend saves BoardDocument snapshot
  -> Board export creates reference PNG and optional mask PNG
  -> Composer receives board_export_asset_id and prompt
  -> PsyPic BFF treats export as normal image/mask input
  -> ImageService calls /v1/images/edits
  -> VersionGraph records board snapshot and output assets
```

### 流式 partial preview

```text
Frontend opens SSE request
  -> PsyPic BFF calls Sub2API with stream=true
  -> Sub2API streams image_generation.partial_image
  -> PsyPic normalizes events
  -> Frontend updates preview
  -> Final event persists task and usage
```

## 存储建议

MVP：

- IndexedDB 保存历史、图片索引和本地预览缓存。
- PsyPic BFF 仍必须负责 session、Sub2API 密钥代持、参数校验、限流和转发。
- 后端可暂不保存图片文件和完整任务正文，但必须保存导入绑定，或在 HttpOnly session 中保存加密绑定状态。
- 优点：历史和图片存储开发快，仍保留密钥不落前端的安全边界。
- 缺点：跨设备不可同步，清缓存会丢本地历史。

正式版：

- PostgreSQL/MySQL 保存任务。
- S3/R2/MinIO 保存图片。
- Redis 保存短期 ticket、任务锁、限流。
- 优点：跨设备、可分享、可审计。
- 缺点：开发量更大。

推荐路线：

- `v0.1-v0.4` 可以先 IndexedDB + BFF，BFF 必须保存 session、导入绑定和必要审计。
- `v0.5` 为流式和任务队列引入轻量服务端任务表。
- `v0.7` 开始引入服务端历史、素材库和对象存储。

## 部署建议

- 前端：Vercel、Cloudflare Pages、Nginx 静态站均可。
- BFF：独立 Node.js / Go / Hono / Fastify 服务。
- Redis：ticket、限流、队列。
- 对象存储：Cloudflare R2、S3、MinIO。

生产注意：

- 不在生产业务服务器构建 Docker 镜像。
- 生产机只允许拉取预构建镜像、重启服务、健康检查、回滚。
- 上线前记录旧镜像 tag 和回滚命令。
