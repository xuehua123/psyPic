# 13. 数据模型草案

## 设计原则

- 密钥、session、导入绑定必须后端保存。
- MVP 历史和图片预览可以先放 IndexedDB，但服务端仍要保存必要的导入绑定、审计和任务摘要。
- `v0.7` 后服务端保存完整任务、素材库和对象存储索引。
- `v0.9` 建立社区最小发布模型。
- `v1.0` 完整社区信息流、互动和审核闭环。

## 存储分层

| 数据 | MVP | 正式版 |
| --- | --- | --- |
| session | Redis / DB | Redis + DB |
| key binding | DB，密文保存 | DB，密文保存 |
| image task | IndexedDB + 服务端摘要 | DB |
| temp asset | BFF 临时资产代理，支持结果下载 | 升级为 image asset |
| image asset | `v0.7` 前不做完整素材库 | DB + Object Storage |
| template | 静态 JSON / DB | DB |
| community work | 不做完整社区 | DB + Object Storage |
| like/favorite/report | 不做完整社区 | DB |
| audit log | 服务端必须有基础记录 | DB / 日志系统 |

## 实体关系

```text
user
  -> session
  -> key_binding
  -> image_task
      -> image_asset
      -> community_work
  -> template
  -> favorite
  -> report

community_work
  -> image_asset
  -> template
  -> like
  -> favorite
  -> report
```

## User

来源可以是 Sub2API 导入，也可以后续扩展 PsyPic 自有账号。

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | PsyPic 用户 id |
| `sub2api_user_id` | string? | Sub2API 用户 id |
| `display_name` | string? | 展示名 |
| `avatar_url` | string? | 头像 |
| `role` | enum | `user/admin` |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

MVP 可用匿名导入用户，但必须能绑定 session 和 key binding。

## Session

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | opaque session id |
| `user_id` | string | 用户 id |
| `key_binding_id` | string? | 当前默认 key binding |

Redis 可保存 session 热数据，DB 保存可审计记录。

## KeyBinding

Sub2API key 不返回前端。

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | binding id |
| `user_id` | string | 用户 id |
| `sub2api_base_url` | string | Sub2API base URL |
| `sub2api_api_key_ciphertext` | text? | 加密 API Key |
| `sub2api_api_key_id` | string? | Sub2API key id |
| `default_model` | string | 默认 `gpt-image-2` |
| `enabled_models` | json | 可用模型 |
| `limits` | json | max n、尺寸、moderation 等 |
| `status` | enum | `active/disabled/expired` |
| `created_at` | datetime | 创建时间 |
| `updated_at` | datetime | 更新时间 |

约束：

- `sub2api_api_key_ciphertext` 和 `sub2api_api_key_id` 至少一个存在。
- 加密密钥来自 `KEY_ENCRYPTION_SECRET`。
- 日志不得输出密文字段。

## ImportTicket / ImportCode

短期导入凭证可以优先放 Redis。

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `code` | string | 一次性导入 code |
| `sub2api_user_id` | string | 来源用户 |
| `sub2api_api_key_id` | string | 来源 key |
| `payload` | json | base_url、limits、model 等 |
| `expires_at` | datetime | 过期时间 |
| `consumed_at` | datetime? | 消费时间 |

要求：

- TTL 30-60 秒。
- 一次性消费。
- `no-store`。
- 不包含明文长期 API Key，除非仅在服务端 Redis 短期密文保存。

## ImageTask

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | task id |
| `user_id` | string | 用户 id |
| `key_binding_id` | string | key binding |
| `type` | enum | `generation/edit/mask/stream/same` |
| `status` | enum | `queued/running/succeeded/failed/canceled` |
| `prompt` | text | prompt，可按隐私策略加密或脱敏 |
| `params` | json | size、quality、format、n 等 |

| `error_code` | string? | 错误码 |

MVP：

- IndexedDB 存完整本地历史。
- 服务端至少可存任务摘要、request id、状态、成本，便于排错和审计。

## TempAsset

MVP 为了支持结果下载和 `/api/assets/{asset_id}`，必须有临时资产代理。它只解决生成结果的短期访问，不等同于正式素材库。

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | temp asset id |
| `user_id` | string | 所属用户 |
| `task_id` | string | 来源任务 |
| `mime_type` | string | image/png 等 |
| `width` | int? | 宽 |
| `height` | int? | 高 |
| `size_bytes` | int? | 文件大小 |
| `temp_storage_key` | string | 临时文件、临时对象或 Redis blob key |
| `expires_at` | datetime | 过期时间 |
| `created_at` | datetime | 创建时间 |

规则：

- `v0.4` 生成接口可以返回 `/api/assets/{temp_asset_id}`。
- `GET /api/assets/{asset_id}` 先查 TempAsset，再查正式 ImageAsset。
- TempAsset 只能本人访问，除非后续生成分享链接。
- TTL 建议 24-72 小时，用户下载和本地 IndexedDB 历史不依赖长期服务端保存。
- `v0.7` 引入正式 ImageAsset 后，成功生成结果可迁移到对象存储和服务端素材库。

## ImageAsset

`v0.7` 后用于服务端历史、素材库、相册和社区长期访问。

字段：

| 字段 | 类型 | 说明 |
| --- | --- | --- |
| `id` | string | asset id |
| `user_id` | string | 所属用户 |
| `task_id` | string? | 来源任务 |
| `storage_key` | string | 对象存储 key |
| `mime_type` | string | image/png 等 |
| `width` | int | 宽 |
| `height` | int | 高 |
| `size_bytes` | int | 文件大小 |
| `sha256` | string | 去重 hash |
| `visibility` | enum | `private/unlisted/public` |
| `created_at` | datetime | 创建时间 |
| `deleted_at` | datetime? | 删除时间 |

规则：

- 私有资产默认不公开访问。
- 公开社区作品也不等于原始素材公开。
- 下载使用短期签名 URL 或 BFF 代理。

## IndexedDB 草案

MVP 本地表：

```text
local_tasks
local_assets
local_settings
local_templates_cache
```

`local_tasks`：

- `id`
- `prompt`
- `params`
- `thumbnail_blob`
- `asset_blob_refs`
- `status`
- `created_at`

本地数据只用于体验，不作为计费、审计和安全依据。

## 迁移顺序

1. `v0.2`：User、Session、KeyBinding、AuditLog。
2. `v0.3-v0.4`：ImageTask 摘要、BillingUsage 摘要。
3. `v0.4`：TempAsset，支撑 `/api/assets/{asset_id}` 和 MVP 结果下载。
4. `v0.5`：Job/Task 状态扩展。
5. `v0.7`：ImageAsset、服务端历史和对象存储。
6. `v0.8`：Template。
7. `v0.9`：CommunityWork、CommunityReport。
8. `v1.0`：CommunityInteraction、精选、完整信息流索引。

## 索引建议

- `session.user_id`
- `key_binding.user_id,status`
- `image_task.user_id,created_at`
- `image_task.status,created_at`
- `temp_asset.user_id,expires_at`
- `image_asset.user_id,created_at`
- `community_work.visibility,moderation_status,published_at`
- `community_work.scene,published_at`
- `community_interaction.work_id,type`
- `community_report.status,created_at`
