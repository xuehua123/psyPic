# 15. 社区审核与隐私策略

## 核心原则

PsyPic 面向商业场景，社区发布必须默认保护用户素材和客户信息。

默认策略：

- 作品默认私有。
- 公开社区必须用户主动选择并二次确认。
- prompt 默认隐藏。
- 参数默认可由用户选择是否公开。
- 参考图和原始素材默认隐藏。
- 拼图画布快照、导出 reference、mask 和文字 brief 默认隐藏。
- 管理员必须能关闭公开发布。
- 管理员必须能下架作品。
- 社区作品应继承当前 project / session 的隐私默认值，除非用户显式提升公开范围。

## 发布可见性

| visibility | 含义 | 访问 |
| --- | --- | --- |
| `private` | 私有 | 仅本人 |
| `unlisted` | 链接可见 | 拥有链接的人 |
| `public` | 公开社区 | 社区可浏览 |

默认值：`private`。

从 `private/unlisted` 切换到 `public` 时必须二次确认。

## 公开内容开关

发布时用户可控制：

- 是否公开 prompt。
- 是否公开生成参数。
- 是否公开参考图。
- 是否公开拼图画布快照或导出图。
- 是否允许同款生成。
- 是否允许作为参考图继续编辑。

默认：

```json
{
  "disclose_prompt": false,
  "disclose_params": false,
  "disclose_reference_images": false,
  "disclose_board_snapshot": false,
  "allow_same_generation": true,
  "allow_reference_reuse": false
}
```

如果用户不公开 prompt，同款生成只能基于模板、公开参数和系统生成的摘要，不展示原始 prompt。

如果作品来自某个工作台版本节点，公开范围和内容开关要和该节点的参数快照一起保存，避免后续恢复时语义漂移。

如果作品来自 Board Mode，默认只公开最终作品图。board snapshot、导出 reference、mask、annotation text 和 source asset ids 都属于创作过程信息，只有用户显式勾选才可公开。

## 发布确认文案

公开社区前必须提示：

```text
确认公开发布？

公开后，其他用户可能看到这张图片，并可基于你允许公开的信息进行同款生成。
请确认图片不包含客户保密素材、未发布商品、个人隐私、商标侵权、肖像权风险或不允许公开的参考图。
```

用户必须勾选：

- 我确认有权公开这张图片。
- 我理解 prompt、参数和参考图的公开设置。

## 审核状态

| 状态 | 含义 |
| --- | --- |
| `draft` | 草稿 |
| `pending` | 待审核 |
| `approved` | 已通过 |
| `rejected` | 拒绝公开 |
| `taken_down` | 已下架 |

推荐策略：

- MVP 不做完整社区。
- `v0.9` 支持审核开关。
- 开启审核时，公开作品先进入 `pending`。
- 关闭审核时，公开作品可直接 `approved`，但仍可举报和下架。
- 审核记录建议保留 project / session / version node 只读上下文，便于回溯来源。

## 举报类型

| reason | 场景 |
| --- | --- |
| `copyright` | 版权或商标 |
| `privacy` | 隐私或客户素材泄露 |
| `sensitive` | 敏感内容 |
| `spam` | 广告、导流、低质刷屏 |
| `impersonation` | 冒充个人或品牌 |
| `other` | 其他 |

举报处理：

- 用户提交举报。
- 作品 `report_count` 增加。
- 达到阈值可自动隐藏，等待管理员处理。
- 管理员可驳回、下架、恢复、精选取消。

## 管理员能力

必须支持：

- 查看公开作品。
- 查看待审核作品。
- 查看举报列表。
- 下架作品。
- 恢复作品。
- 设置精选。
- 取消精选。
- 关闭公开发布。
- 关闭同款生成。
- 查看作品关联的用户、任务、request id 和 key binding。
- 查看作品所属 project / session / version node，但不得自动展开未公开参考图。

管理员界面不得默认展示：

- 明文 API Key。
- 用户完整 cookie。
- 未公开的原始参考图。
- 用户选择隐藏的 prompt，除非有明确审计权限。

## 同款生成隐私规则

同款生成必须遵守发布者设置：

| 设置 | 同款行为 |
| --- | --- |
| prompt 公开 | 可使用原 prompt 作为草稿 |
| prompt 隐藏 | 只能使用模板和摘要生成新 prompt |
| 参数公开 | 可复用尺寸、质量、格式等 |
| 参数隐藏 | 使用模板默认参数 |
| 参考图公开 | 可作为参考图 |
| 参考图隐藏 | 不暴露原参考图 |

同款生成产生的是新任务，必须归属于当前用户，并重新计费。

## 商业风险提示

以下场景发布前需要强提示：

- 客户商品未公开。
- 图片含清晰人脸。
- 图片含商标、品牌 logo。
- 图片含内部包装、价格、合同、二维码。
- 图片来自第三方摄影或素材站。
- prompt 中包含客户名称、活动策略、投放计划。

## 保留与删除

删除作品：

- 默认隐藏社区作品。
- 不一定删除原任务和私有资产。
- 用户要求删除资产时，异步删除对象存储文件。

下架作品：

- 社区不可见。
- 作者仍可在个人历史中看到。
- 管理员保留审计记录。

删除用户：

- 私有内容按用户要求删除。
- 公开作品可按策略匿名化或删除。
- 审计日志可按合规要求保留必要字段。

## 日志与审计

记录：

- 发布、编辑、公开、取消公开。
- 举报、处理、下架、恢复。
- 精选、取消精选。
- 同款生成来源 work id。

不记录：

- 明文 API Key。
- 完整 cookie。
- 未脱敏敏感 prompt。
- 原始图片二进制。

## 上线前检查

- 公开发布默认关闭还是开启。
- 是否默认审核。
- 举报阈值。
- 管理员审计权限范围。
- 用户删除作品是否物理删除图片。
- 是否允许匿名/手动 key 用户公开发布。
- 是否需要用户协议和社区规范页面。
