# 17. 测试与验收用例

## 目标

本文件用于开发阶段验证 PsyPic 是否真的可用，而不是只完成 UI 或接口草稿。

覆盖：

- Sub2API 一键导入。
- session 和密钥安全。
- 文生图。
- 图生图。
- TempAsset 下载。
- IndexedDB 基础历史。
- 参数校验。
- 错误处理。
- 商业尺寸预设。
- MVP/V1/封板验收。

## 测试类型

| 类型 | 工具 | 阶段 |
| --- | --- | --- |
| 单元测试 | Vitest | 每个阶段 |
| API 测试 | Vitest / Supertest / fetch | `v0.2+` |
| 前端组件测试 | Testing Library | `v0.1+` |
| E2E | Playwright | `v0.3+` |
| 手工验收 | 浏览器 | 每个阶段 |
| 安全检查 | 脚本 + 人工检查 | `v0.2+` |

## 基础命令

```bash
pnpm lint
pnpm typecheck
pnpm test
pnpm build
```

E2E：

```bash
pnpm e2e
```

首次运行 E2E 前安装 Chromium：

```bash
pnpm e2e:install
```

当前 `pnpm e2e` 使用受保护的本地 E2E session 入口和 fake Sub2API，上线前仍需用真实 Sub2API、对象存储和 staging 数据库补一次人工验收。

## API 测试用例

### Health

请求：

```http
GET /api/health
```

期望：

- HTTP 200。
- `data.ok=true`。
- 返回 db、redis、temp_asset 状态。
- `storage` 只在配置 R2/S3/MinIO 时返回；未配置时应为 `skipped` 或不返回，不能阻塞 MVP。

### Import Exchange 成功

请求：

```http
POST /api/import/exchange
Content-Type: application/json

{
  "import_code": "valid_one_time_code"
}
```

期望：

- HTTP 200。
- 返回 `session_bound=true`。
- 设置 `psypic_session` HttpOnly cookie。
- 响应体不包含 `api_key`。
- 响应头包含 `Cache-Control: no-store`。

### Import Exchange 重复消费

步骤：

1. 使用同一个 `import_code` 请求成功一次。
2. 再次使用同一个 `import_code` 请求。

期望：

- 第二次 HTTP 400 或 403。
- 错误码为 `invalid_import_code` 或 `forbidden`。
- 不创建新的 session。

### Session

请求：

```http
GET /api/session
Cookie: psypic_session=...
```

期望：

- HTTP 200。
- 返回 `authenticated=true`。
- 返回 binding、limits、features。
- 不返回 API Key。

### 文生图成功

请求：

```http
POST /api/images/generations
Content-Type: application/json

{
  "prompt": "为一瓶透明玻璃香水生成高端电商主图...",
  "model": "gpt-image-2",
  "size": "1024x1024",
  "quality": "medium",
  "n": 1,
  "output_format": "png",
  "background": "auto",
  "moderation": "auto"
}
```

期望：

- HTTP 200。
- 返回 `task_id`。
- 返回至少 1 个 `images[]`。
- 每个 image 有 `asset_id` 和 `/api/assets/{asset_id}` URL。
- 返回 `request_id`。
- 如果 Sub2API 返回 upstream request id，响应中透出。
- 记录 usage 或 estimated cost。

### 文生图参数错误

场景：

- prompt 为空。
- `n` 超过 limits。
- `background=transparent`。
- 非法 size。

期望：

- HTTP 400。
- 错误码 `invalid_parameter`。
- `details.field` 指向错误字段。
- 不调用 Sub2API。

### 图生图成功

请求：

```http
POST /api/images/edits
Content-Type: multipart/form-data

prompt=保持产品主体完全一致...
image=<product.png>
size=1024x1024
quality=medium
n=1
```

期望：

- HTTP 200。
- 返回 `task_id` 和图片。
- 上传文件被校验。
- 响应可下载。

### 遮罩编辑成功

`v0.6` 起测试。

请求：

```http
POST /api/images/edits
Content-Type: multipart/form-data

prompt=只替换涂抹区域...
image=<product.png>
mask=<mask.png>
size=1024x1024
quality=medium
n=1
```

期望：

- HTTP 200。
- `mask` 是 PNG 文件。
- BFF 转发给 Sub2API 的 multipart 同时包含 `image` 和 `mask`。
- 响应不泄露 API Key。

### 图生图上传错误

场景：

- 未上传 image。
- 上传非图片。
- 上传超大文件。
- 上传非 PNG mask。
- 上传尺寸与参考图不一致的 mask。

期望：

- 未上传 image：HTTP 400。
- 非图片：HTTP 415。
- 超大文件：HTTP 413。
- 非 PNG mask：HTTP 415，`details.field=mask`。
- mask 尺寸不一致：HTTP 400，`details.field=mask`。
- 错误提示可读。

### 任务状态查询

`v0.5` 起测试。

请求：

```http
GET /api/tasks/{task_id}
Cookie: psypic_session=...
```

期望：

- 本人访问 HTTP 200。
- 返回 `id`、`type`、`status`、`prompt`、`params`、`created_at`、`updated_at`。
- 成功任务返回 images、usage、duration_ms。
- 失败任务返回脱敏后的 `error.code` 和 `error.message`。
- 非本人或不存在任务返回 404。
- 未登录返回 401。

### 任务取消

`v0.5` 起测试。

请求：

```http
POST /api/tasks/{task_id}
Cookie: psypic_session=...
```

期望：

- queued/running 任务返回 HTTP 200 且 `status=canceled`。
- succeeded/failed/canceled 任务返回当前状态，不删除结果。
- 只能取消当前 session 所属用户的任务。
- 前端任务状态区显示取消后的状态，并提供重新生成入口。

### 图片任务并发限制

`v0.5` 起测试。

场景：

- 同一用户已有 `queued` 或 `running` 图片任务。
- 再次请求文生图、图生图或流式文生图。

期望：

- HTTP 429。
- 错误码 `rate_limited`。
- 错误提示说明当前已有任务正在运行。
- 不调用 Sub2API。
- 默认限制为每用户 1 个 active 图片任务，可通过 `PSYPIC_MAX_ACTIVE_IMAGE_TASKS_PER_USER` 调整。

### 服务端历史

`v0.7` 起测试。

请求：

```http
GET /api/history?limit=30&cursor=task_xxx
Cookie: psypic_session=...
```

期望：

- 未登录返回 HTTP 401，错误码 `unauthorized`。
- 只返回当前用户已成功并且有图片的任务。
- 历史项包含 `task_id`、`type`、`prompt`、`params`、`thumbnail_url`、`images`、`usage`、`created_at`。
- 如果任务有 upstream request id，历史项透出 `upstream_request_id`。
- `next_cursor` 使用当前页最后一条 `task_id`，下一页不跳过记录。
- 响应不包含 API Key、session cookie 或 key binding secret。

### 素材库

`v0.7` 起测试。

请求：

```http
GET /api/library?limit=30&favorite=true&tag=电商主图&q=香水
Cookie: psypic_session=...
```

期望：

- 未登录返回 HTTP 401，错误码 `unauthorized`。
- 只返回当前用户已成功任务中的图片资产。
- 素材项包含 `asset_id`、`task_id`、`prompt`、`url`、`thumbnail_url`、`favorite`、`tags`、`created_at`。
- `favorite=true` 只返回收藏资产。
- `tag` 能按规范化标签过滤。
- `q` 能匹配 prompt、task id、asset id 和标签。
- 响应不包含 API Key、session cookie 或 key binding secret。

请求：

```http
PATCH /api/library/{asset_id}
Content-Type: application/json
Cookie: psypic_session=...

{
  "favorite": true,
  "tags": ["电商主图", "香水"]
}
```

期望：

- 本人资产返回 HTTP 200，并返回更新后的素材项。
- tags 去重、去空白。
- 非本人或不存在资产返回 404。
- `favorite` 非布尔值或 `tags` 非字符串数组时返回 HTTP 400。

### 创作台社区发布入口

`v0.9` 起测试。

步骤：

2. 在素材卡片点击发布作品。

## 前端 E2E 用例

### 首屏创作台

步骤：

1. 打开 `/`。
2. 查看首屏。

期望：

- 默认进入创作台。
- 左侧参数栏可见。
- 中间结果区可见。
- 右侧历史/详情区可见。
- 商业模板入口可见。
- 没有营销 hero 抢占首屏。

### 画廊工作台与版本流

步骤：

1. 在创作台输入 Prompt 并生成。
2. 修改 Prompt 或尺寸后再次生成。
3. 在版本流点击第一条节点的恢复参数。
4. 点击第一条节点的从此分叉，再输入新 Prompt 生成。

期望：

- 中间主区以结果画廊为视觉核心，能容纳多版本图片。
- 每次生成后版本流新增节点，节点展示 prompt 摘要和参数摘要。
- 恢复参数只恢复 Composer 和参数控件，不删除任何历史节点。
- 从旧节点分叉生成时，旧后续节点保留，新节点显示为新的分支。
- 分支图能展示主线和分支关系。
- Inspector 能展示当前节点的 request id、耗时和参数快照。

### 手动配置并生成

步骤：

1. 打开设置。
2. 输入 Sub2API Base URL 和 API Key。
3. 回到创作台。
4. 输入测试 prompt。
5. 点击生成。

期望：

- 按钮进入 loading。
- 生成成功后展示结果卡。
- 可下载图片。
- 历史区出现记录。

### 服务端素材库同步

- 点击收藏调用 `PATCH /api/library/{asset_id}`。
- 收藏状态更新后按钮变为取消收藏。
- 素材库不可用时仍保留本地历史继续编辑。

## 安全验收

### API Key 不进 URL

检查：

- 浏览器地址栏。
- Network request URL。
- server log。
- client log。

期望：

- 不出现明文 API Key。
- 不出现长期 bearer token。

### API Key 不进前端长期明文存储

检查：

- localStorage。
- sessionStorage。
- IndexedDB。
- cookie 非 HttpOnly 内容。

期望：

- 不出现明文 API Key。

### 导入响应不返回 API Key

检查：

- `/api/import/exchange` 响应体。
- DevTools Network response。

期望：

- 不包含 `api_key`。
- 不包含 `Authorization`。

### 日志脱敏

检查：

- 生成请求日志。
- 导入请求日志。
- 错误日志。

期望：

- `Authorization` 被脱敏。
- `psypic_session` 被脱敏。
- key binding 不输出密文字段。

## 错误处理验收

| 场景 | 期望 |
| --- | --- |
| prompt 为空 | 前端阻止提交，或后端 400 |

| key 失效 | 提示检查 Sub2API key |
| 额度不足 | 提示额度不足 |
| 429 | 提示稍后重试 |
| 超时 | 提示任务超时，可重试 |

| 上传过大 | 提示最大文件大小 |

### 暗色模式（Phase 6）

- AppTopNav 工具区出现亮 / 暗 / 跟随系统三态切换按钮，aria-label 含
  当前态 + 下一态。
- 三态循环：light → dark → system → light。
- 切换 dark：`<html>` 加 `class="dark"`，CSS var 集中翻成暗色 token，
  全站 surface / text / border 不留亮色硬编码。

- 用户选择持久化到 localStorage（key `psypic-theme`），刷新后保持。
- 暗色用户首次进入页面：`<head>` 内嵌的 `ThemeNoFlashScript` 在 React
  hydrate 之前同步把 class 写到 `<html>`，无白屏一闪。

pnpm test tests/components/theme.test.tsx
