# 00. 调研总结

## OpenAI `gpt-image-2`

截至 2026-04-30，OpenAI 官方文档显示：

- Model ID：`gpt-image-2`
- 当前快照：`gpt-image-2-2026-04-21`
- 定位：OpenAI 当前最强 GPT Image 生成与编辑模型。
- 能力：高质量图片生成、图片编辑、灵活尺寸、高保真图片输入。
- 可能要求：组织完成 OpenAI API Organization Verification。

## 可用 API

### Images API

适合 PsyPic MVP，因为它直接、稳定、参数明确。

- `POST /v1/images/generations`：文生图。
- `POST /v1/images/edits`：图片编辑、参考图生成、遮罩局部编辑。

### Responses API

适合 V1+ 的多轮编辑、上下文创作、对话式改图。

- 外层 `model` 使用文本/多模态主模型，例如 `gpt-5.5`、`gpt-5` 及更新模型。
- 工具使用 `tools: [{ "type": "image_generation", ... }]`。
- 不应把 `gpt-image-2` 当成 Responses API 外层 `model`。

## 关键官方能力

- 支持尺寸 `auto` 和大量自定义合规尺寸。
- 支持 `quality: auto | low | medium | high`。
- 支持 `output_format: png | jpeg | webp`。
- 支持 JPEG/WebP 的 `output_compression: 0-100`。
- 支持 `moderation: auto | low`。
- 支持流式生成：`stream: true`。
- 支持 `partial_images: 0-3`。
- `gpt-image-2` 不支持透明背景，`background: "transparent"` 会失败。
- `gpt-image-2` 自动使用高保真图片输入，不应传 `input_fidelity`。

## 尺寸规则

`gpt-image-2` 支持灵活尺寸，但需要满足：

- 最大边不超过 `3840px`。
- 宽高都必须是 `16px` 的倍数。
- 长边 / 短边不超过 `3:1`。
- 总像素不少于 `655,360`，不超过 `8,294,400`。
- 超过 `2560x1440` 总像素的 2K+ 输出被官方标注为实验性范围。

常用预设：

- `auto`
- `1024x1024`
- `1536x1024`
- `1024x1536`
- `2048x2048`
- `2048x1152`
- `3840x2160`
- `2160x3840`

## 成本与延迟

官方说明复杂 prompt 可能最多接近 2 分钟。成本由三部分组成：

- prompt 文本输入 token。
- 编辑/参考图的图片输入 token。
- 图片输出 token。

`partial_images` 每张会额外产生 100 image output tokens。PsyPic 必须把流式预览作为高级体验，而不是无成本预览。

官方示例成本：

| 尺寸 | low | medium | high |
| --- | ---: | ---: | ---: |
| `1024x1024` | $0.006 | $0.053 | $0.211 |
| `1024x1536` | $0.005 | $0.041 | $0.165 |
| `1536x1024` | $0.005 | $0.041 | $0.165 |

注：这些是官方示例估算，最终应以响应 `usage` 和最新定价为准。

## Prompt 经验

官方 prompting guide 建议：

- 用稳定结构：背景/场景 -> 主体 -> 关键细节 -> 约束。
- 明确用途：广告图、UI mock、海报、信息图、头像等。
- 写实图直接使用 `photorealistic`。
- 图中需要文字时，把文字放进引号，说明字体、颜色、位置、大小。
- 图片编辑时使用“只修改 X，保持 Y 不变”。
- 多参考图时明确编号：Image 1 是主体，Image 2 是风格参考。

## 参考项目：CookSleep/gpt_image_playground

参考项目地址：https://github.com/CookSleep/gpt_image_playground

值得借鉴：

- Images API 和 Responses API 双模式。
- 文生图、参考图、遮罩编辑。
- 支持最多 16 张参考图。
- 自定义尺寸规整与官方尺寸限制校验。
- 任务详情中展示实际参数、耗时、改写后的 prompt。
- 本地 IndexedDB 历史记录。
- 结果转参考图继续迭代。
- URL 参数快速填入 `apiUrl`、`apiKey`。

生产站不建议照搬：

- 不建议把密钥放在 URL 查询参数里。
- 不建议只做纯前端直连生产 API，因为 CORS、密钥暴露、额度风控都会变差。
- PsyPic 应采用后端/BFF 代理和 Sub2API 短期 ticket 导入。

## 竞品启发

以 LiblibAI、Krea、Recraft、Leonardo、Freepik 等 AI 生图站为参考，PsyPic 应重点学习这些产品体验：

- 第一屏就是创作台，不是营销页。
- 参数面板要清晰但不压迫主画布。
- 模板和风格预设降低用户 prompt 门槛。
- 历史/素材/收藏是核心，不只是附属功能。
- 结果图必须能快速再编辑、作为参考图、批量下载。
- 高级功能应被折叠到“高级设置”，避免 MVP 首屏复杂。
