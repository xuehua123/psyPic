# 05. 交互设计方案

## 设计原则

- 第一屏就是可用创作台，不做营销首页。
- 默认参数简单，高级参数折叠。
- 生成结果是视觉中心，参数面板服务结果，不抢主视线。
- 所有结果都能继续编辑、转参考图、下载、收藏。
- 移动端不是缩水版，核心生成流程必须完整可用。

## 桌面端布局

推荐三栏布局：

```text
Left Panel       Center Workspace              Right Panel
参数/模板        Prompt + 结果网格/画布         历史/详情/素材
```

### 左侧参数栏

包含：

- 模式：文生图 / 图生图 / 局部编辑。
- 模型：默认 `gpt-image-2`。
- 尺寸：预设 + 自定义。
- 质量：草图 / 标准 / 高质量 / 自动。
- 输出格式：PNG / JPEG / WebP。
- 数量。
- 高级设置：compression、moderation、stream、partial images。

### 中间创作区

包含：

- Prompt 输入框。
- 快捷模板入口。
- 参考图拖拽区。
- 生成按钮。
- 结果网格。
- 流式 partial preview。
- 失败状态和重试。

### 右侧历史/详情

默认显示最近任务。

点击任务后显示：

- 原始 prompt。
- 实际参数。
- usage。
- 耗时。
- request id。
- 错误信息。
- “继续编辑”按钮。

## 移动端布局

推荐：

- 顶部：站点名、设置、历史。
- 中间：Prompt、结果。
- 底部：固定生成按钮。
- 参数面板使用底部抽屉。
- 历史使用全屏抽屉。
- 图片详情使用全屏预览。

## 创作流程

### 文生图

1. 用户输入 prompt。
2. 选择模板或风格。
3. 选择尺寸/质量。
4. 点击生成。
5. 显示 partial 或 loading。
6. 展示结果。
7. 用户下载、收藏或继续编辑。

### 图生图

1. 用户上传参考图。
2. 输入改图说明。
3. 选择“保持主体/换背景/换风格”等模板。
4. 生成。
5. 结果可作为下一轮参考图。

### 局部编辑

1. 用户选择主图。
2. 打开遮罩编辑器。
3. 涂抹要修改区域。
4. 输入局部修改 prompt。
5. 生成。
6. 对比原图和结果。

## 模板设计

模板不是静态 prompt，而是结构化表单。模板优先服务商业场景，再逐步开放社区同款复用。

示例：电商主图模板

- 产品类型。
- 背景风格。
- 光线。
- 构图。
- 是否留白。
- 是否添加文字。

生成 prompt 时组合成结构化描述。

核心商业模板：

- 电商主图。
- 产品换背景。
- 商品场景图。
- 广告 Banner。
- 小红书/公众号/短视频封面。
- 品牌海报。
- 人像头像。
- 应用/网站首屏视觉。

## 灵感社区

社区不是独立娱乐流，而是创作台的上游入口。

### 信息流

- 按商业场景分区，而不是只按风格分区。
- 卡片展示成图、场景标签、尺寸、用途、作者和同款入口。
- 支持精选、最新、热门、收藏、我的关注。

### 作品详情

- 展示成图、用途、参数摘要、使用模板、作者说明。
- Prompt、参考图和参数默认由发布者选择是否公开。
- 提供“一键同款”“作为参考图”“加入项目”“收藏模板”。

### 发布流程

- 从结果卡片发布。
- 选择公开范围：私有、链接可见、公开社区。
- 选择是否公开 prompt、参数、参考图和原始素材。
- 发布前做内容安全和隐私提醒。

## Prompt 助手

功能：

- 中文优化。
- 中文转英文。
- 结构化 prompt。
- 加入构图/光线/材质。
- 加入“不改变主体”的约束。
- 为图片编辑生成保留项。

示例结构：

```text
Scene:
Subject:
Style:
Lighting:
Composition:
Text:
Constraints:
```

## 图片结果卡片

每张图必须有：

- 预览。
- 下载。
- 收藏。
- 复制 prompt。
- 继续编辑。
- 作为参考图。
- 查看详情。

批量选择时：

- 下载 ZIP。
- 删除。
- 加标签。
- 收藏。

## 状态设计

### 空状态

不要写大量说明。给用户可直接点击的模板：

- 产品主图。
- 头像。
- 海报。
- 插画。
- UI mockup。

### 加载状态

- 显示任务进度。
- 有 partial image 时展示 partial。
- 无 partial 时显示稳定占位。
- 提供取消按钮。

### 错误状态

错误要给动作：

- 参数错误：跳到对应参数。
- 额度不足：打开 Sub2API 额度页面。
- key 无效：重新导入。
- 超时：重试或降低质量。

## 视觉方向

PsyPic 应该是专业创作工具，不是花哨 landing page。

建议：

- 低噪声深浅双主题。
- 图片占主导。
- 参数控件紧凑、清晰。
- 卡片半径不超过 8px。
- 按钮使用图标 + tooltip。
- 避免大面积单一紫蓝渐变。
- 不使用装饰性光斑、渐变球。
