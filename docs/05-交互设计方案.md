# 05. 交互设计方案

## 设计原则

- 第一屏就是可用创作台，不做营销首页。
- 默认参数简单，高级参数折叠。
- 生成结果是视觉中心，参数面板服务结果，不抢主视线。
- 所有结果都能继续编辑、转参考图、下载、收藏。
- 移动端不是缩水版，核心生成流程必须完整可用。

## 桌面端布局

推荐工作台式布局，而不是大表单页面：

```text
Project Sidebar      Main Canvas / Transcript      Inspector + Task Dock
项目、会话、分支      对话、结果、版本、Board Mode      参数、图层、模板、素材、任务、搜索
```

### Project Sidebar

包含：

- 项目列表。
- 当前项目下的会话 / 分支。
- 新建项目。
- 从历史节点分叉。
- 导入素材入口。
- 灵感社区和设置入口。

### Main Canvas / Transcript

包含：

- Prompt Composer。
- 对话式版本流。
- 生成结果消息。
- 版本节点卡片。
- Board Mode。
- 参考图和局部编辑预览。
- 流式 partial preview。
- 失败状态、重试和继续编辑。

### Board Mode（多素材多图层画布）

Board Mode 是 Main Canvas 的一种可切换模式，用于把多张素材、历史结果、手绘标注、文字 brief 和 mask 草图组合成可提交给 AI 编辑的视觉输入。

核心交互：

- 从素材库、历史结果、结果卡或社区同款草稿把图片加入画布。
- 图片层支持移动、缩放、旋转、复制、删除、锁定、隐藏和层级排序。
- 手绘层支持画笔、橡皮、颜色、粗细，用于构图标注或局部修改说明。
- mask 层与普通标注层分离，可导出 alpha PNG 进入 `/api/images/edits` 的 `mask` 字段。
- 画布可扁平化导出 PNG，并一键作为图生图参考图。
- 每次导出或提交都保存 board snapshot，并关联到 version node。
- 素材库可以按 `background / subject / decoration / text / text_effect / texture_light / shape / brand / reference / mask` 分类筛选后拖入画布。
- 文字默认作为可编辑 TextLayer，不建议直接烘焙进图片结果里。

### Inspector + Task Dock

Inspector 默认显示：

- 模式：文生图 / 图生图 / 局部编辑。
- 模型：默认 `gpt-image-2`。
- 尺寸：预设 + 自定义。
- 质量：草图 / 标准 / 高质量 / 自动。
- 输出格式：PNG / JPEG / WebP。
- 数量。
- 高级设置：compression、moderation、stream、partial images。
- 商业模板、历史、素材和参数快照。

Board Mode 下 Inspector 切换为：

- Layers：图层列表、锁定、隐藏、排序、删除。
- Transform：位置、尺寸、缩放、旋转、不透明度。
- Brush / Mask：画笔、橡皮、颜色、粗细、mask 模式。
- Export：导出 reference PNG、导出 mask PNG、作为参考图编辑、作为局部编辑 mask。
- Snapshot：保存快照、恢复当前版本节点的 board 状态。

Task Dock 默认显示：

- queued / running / partial / failed / canceled 任务。
- 取消、重试、导出和同步状态。
- 批量、ZIP、社区发布等后台动作。
- 当前任务 request id、upstream request id、耗时。

## 移动端布局

推荐：

- 顶部：站点名、设置、历史。
- 中间：对话流、结果和版本节点。
- 底部：固定 Composer + 主生成按钮。
- Project Sidebar 使用左侧抽屉。
- Inspector 和 Task Dock 使用底部抽屉。
- 图片详情使用全屏预览。

移动端不复制桌面三栏，而是保留创作主线，其他能力分层收纳。

## 搜索与命令面板

工作台需要一个全局搜索 / 命令入口，桌面端可用快捷键唤起，移动端可放在顶部工具区。

搜索覆盖：

- 项目、会话和版本节点。
- Prompt、模板、场景、标签、尺寸、格式。
- 资产、相册、社区作品和批量任务。
- request id、task id、upstream request id。

命令覆盖：

- 新建项目。
- 导入素材。
- 从当前节点分叉。
- 打开参数面板。
- 批量导出。
- 打开设置或管理台。

## 创作流程

### 文生图

1. 用户输入 prompt。
2. 选择模板或风格。
3. 选择尺寸/质量。
4. 点击生成。
5. 显示 partial 或 loading。
6. 展示结果。
7. 用户下载、收藏或继续编辑。
8. 新结果进入当前会话的版本流，而不是只落成一张孤立图片。

### 图生图

1. 用户上传参考图。
2. 输入改图说明。
3. 选择“保持主体/换背景/换风格”等模板。
4. 生成。
5. 结果可作为下一轮参考图。
6. 如果从旧版本节点出发，生成结果应显示分支关系。

### 局部编辑

1. 用户选择主图。
2. 打开遮罩编辑器。
3. 涂抹要修改区域。
4. 输入局部修改 prompt。
5. 生成。
6. 对比原图和结果。

### Board Mode 编辑

1. 用户切到 Board Mode。
2. 从素材库、历史结果或结果卡拖入多张图片。
3. 调整位置、缩放、旋转和层级。
4. 用画笔标注修改意图，必要时启用 mask 层。
5. 点击“作为参考图编辑”或“作为局部编辑 mask”。
6. 画布导出 PNG / alpha mask，并写入当前 Composer。
7. 用户补充 prompt 后提交 `/api/images/edits`。
8. 新结果进入当前 session 的版本流，并保存 board snapshot 以便恢复或分叉。

### 版本节点交互

- 每次生成或编辑都会形成一个可回溯版本节点。
- “恢复参数”只写回 Composer 和参数控件，不删除历史节点。
- “从此分叉”会保留原分支，创建新的非破坏性节点。
- “作为参考图”会把当前结果带入下一次编辑，但不覆盖原图。
- “打开 Board Mode”会把当前节点输出加入 Board Mode，board snapshot 与后续导出一起挂到新版本节点。
- 任务状态变化应同步到 Task Dock 和版本流，而不是只靠按钮 loading。

## 模板设计

模板不是静态 prompt，而是结构化表单。模板优先服务商业场景，再逐步开放社区同款复用。

示例：电商主图模板

- 产品类型。
- 背景风格。
- 光线。
- 构图。
- 是否留白。
- 是否添加文字。

生成 prompt 时组合成结构化描述。

模板应用后要同时影响：

- 当前会话的 Composer。
- 当前版本节点的参数快照。
- Inspector 中的默认尺寸、质量、格式和参考图约束。

核心商业模板：

- 电商主图。
- 产品换背景。
- 商品场景图。
- 广告 Banner。
- 小红书/公众号/短视频封面。
- 品牌海报。
- 人像头像。
- 应用/网站首屏视觉。

## 灵感社区

社区不是独立娱乐流，而是创作台的上游入口。

### 信息流

- 按商业场景分区，而不是只按风格分区。
- 卡片展示成图、场景标签、尺寸、用途、作者和同款入口。
- 支持精选、最新、热门、收藏、我的关注。

### 作品详情

- 展示成图、用途、参数摘要、使用模板、作者说明。
- Prompt、参考图和参数默认由发布者选择是否公开。
- 提供“一键同款”“作为参考图”“加入项目”“收藏模板”。
- 同款草稿返回后应落在当前项目 / 会话上下文里，而不是另起一个无关页面状态。

### 发布流程

- 从结果卡片发布。
- 选择公开范围：私有、链接可见、公开社区。
- 选择是否公开 prompt、参数、参考图和原始素材。
- 发布前做内容安全和隐私提醒。

## Prompt 助手

功能：

- 中文优化。
- 中文转英文。
- 结构化 prompt。
- 加入构图/光线/材质。
- 加入“不改变主体”的约束。
- 为图片编辑生成保留项。

Prompt 助手的输出应直接服务于当前会话的 Composer 和版本快照，而不是作为一次性文本结果结束。

示例结构：

```text
Scene:
Subject:
Style:
Lighting:
Composition:
Text:
Constraints:
```

## 图片结果卡片

每张图必须有：

- 预览。
- 下载。
- 收藏。
- 复制 prompt。
- 继续编辑。
- 作为参考图。
- 查看详情。
- 显示所属项目、会话、版本节点和任务状态。

批量选择时：

- 下载 ZIP。
- 删除。
- 加标签。
- 收藏。

## 状态设计

### 空状态

不要写大量说明。给用户可直接点击的模板：

- 产品主图。
- 头像。
- 海报。
- 插画。
- UI mockup。

### 加载状态

- 显示任务进度。
- 有 partial image 时展示 partial。
- 无 partial 时显示稳定占位。
- 提供取消按钮。
- Task Dock 和版本流同步刷新，不要让用户只在按钮上猜进度。

### 错误状态

错误要给动作：

- 参数错误：跳到对应参数。
- 额度不足：打开 Sub2API 额度页面。
- key 无效：重新导入。
- 超时：重试或降低质量。
- 分支冲突或版本节点缺失：回到最近可用节点并提示重新分叉。

## 视觉方向

PsyPic 应该是专业创作工具，不是花哨 landing page。

建议：

- 低噪声深浅双主题。
- 图片占主导。
- 参数控件紧凑、清晰。
- 卡片半径不超过 8px。
- 按钮使用图标 + tooltip。
- 避免大面积单一紫蓝渐变。
- 不使用装饰性光斑、渐变球。
