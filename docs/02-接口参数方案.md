# 02. 接口参数方案

## 默认 API 路线

MVP 默认使用 Images API：

- 文生图：`POST /v1/images/generations`
- 图生图/编辑/遮罩：`POST /v1/images/edits`

V1+ 支持 Responses API：

- `POST /v1/responses`
- 外层模型：`gpt-5.5` 或其他支持 `image_generation` tool 的主模型。
- 工具：`type: "image_generation"`。

## Images API 参数

| 参数 | UI 控件 | 默认值 | 说明 |
| --- | --- | --- | --- |
| `model` | 模型选择 | `gpt-image-2` | MVP 可固定，后续允许管理员配置 |
| `prompt` | 大文本框 | 空 | 必填 |
| `n` | 步进器 | `1` | MVP 建议最大 4 |
| `size` | 预设 + 自定义 | `auto` | 需要前端和后端双重校验 |
| `quality` | 分段控件 | `auto` | `auto/low/medium/high` |
| `output_format` | 分段控件 | `png` | `png/jpeg/webp` |
| `output_compression` | 滑杆 | 空 | 仅 `jpeg/webp` 可用 |
| `background` | 高级选项 | `auto` | `gpt-image-2` 禁止 `transparent` |
| `moderation` | 高级选项 | `auto` | `auto/low` |
| `stream` | 开关 | `false` | `v0.5` 开启 |
| `partial_images` | 步进器 | `0` | 0-3，流式时可用 |
| `image[]` | 上传 | 空 | edits multipart 场景 |
| `images[]` | URL/数据引用 | 空 | edits JSON 场景 |
| `mask` | 遮罩编辑 | 空 | edits 局部编辑 |
| `board_export_asset_id` | 拼图画布导出 | 空 | 工作台上下文，不直接传给上游 |

## 不展示或特殊处理参数

| 参数 | 处理 |
| --- | --- |
| `input_fidelity` | `gpt-image-2` 自动高保真，不展示、不传 |
| `style` | 早期 DALL-E 风格参数，不作为 `gpt-image-2` 主参数 |
| `response_format` | 新图像接口主要返回 base64，前端不暴露 |
| `background=transparent` | 禁用，并显示“gpt-image-2 暂不支持透明背景” |

## 尺寸预设

基础预设：

- `auto`
- `1024x1024`，方图。
- `1536x1024`，横图。
- `1024x1536`，竖图。
- `2048x2048`，2K 方图。
- `2048x1152`，2K 横图。
- `3840x2160`，4K 横图。
- `2160x3840`，4K 竖图。

比例预设：

- `1:1`
- `4:3`
- `3:4`
- `3:2`
- `2:3`
- `16:9`
- `9:16`
- `21:9`
- `9:21`

自定义宽高校验：

- 宽高都必须为正整数。
- 自动规整到 16 的倍数。
- 最大边不超过 3840。
- 长宽比不超过 3:1。
- 总像素在 655,360 到 8,294,400 之间。
- 管理端可限制是否允许 4K。

## 默认参数建议

面向普通用户：

```json
{
  "model": "gpt-image-2",
  "size": "auto",
  "quality": "auto",
  "output_format": "png",
  "moderation": "auto",
  "n": 1
}
```

面向快速草图：

```json
{
  "model": "gpt-image-2",
  "size": "1024x1024",
  "quality": "low",
  "output_format": "jpeg",
  "output_compression": 80,
  "moderation": "auto",
  "n": 1
}
```

面向最终成品：

```json
{
  "model": "gpt-image-2",
  "size": "1536x1024",
  "quality": "high",
  "output_format": "png",
  "moderation": "auto",
  "n": 1
}
```

## 工作台参数快照

所有生成、编辑、同款和批量任务都必须保存参数快照，用于 version node 回溯、任务重试、素材详情、社区发布和成本审计。

快照至少包含：

- `prompt`：用户提交的原始 prompt。
- `rendered_prompt`：模板或 Prompt 助手渲染后的最终 prompt，可选。
- `model` / `provider`：当前为 `gpt-image-2` + Sub2API，后续可扩展到其他 image provider。
- `size` / `quality` / `output_format` / `output_compression` / `moderation` / `n`。
- `commercial_preset` / `template_id` / `template_fields`。
- `source_asset_ids`：参考图、社区同款来源图或历史结果图。
- `mask_asset_id`：局部编辑 mask。
- `board_document_id` / `board_snapshot`：拼图画布来源和提交时快照。
- `board_export_asset_id`：画布扁平化 reference PNG 或 mask PNG 导出资产。
- `project_id` / `session_id` / `parent_version_node_id`：工作台上下文。

规则：

- 恢复参数只写回 Composer 和参数控件，不删除旧结果。
- 从旧版本继续生成必须创建新的 version node，不覆盖旧节点。
- 重试任务必须创建新 task，并引用原 task / version node。
- Board Mode 导出的 reference/mask 只作为标准 `image` / `mask` 进入 edits，上游不感知 BoardDocument。
- 社区发布默认不公开完整参数快照，除非用户主动勾选。

## Responses API 工具参数

用于多轮编辑时：

```json
{
  "model": "gpt-5.5",
  "input": "生成一张产品海报...",
  "tools": [
    {
      "type": "image_generation",
      "model": "gpt-image-2",
      "action": "generate",
      "size": "1024x1024",
      "quality": "medium",
      "output_format": "png"
    }
  ]
}
```

`action` 可选：

- `auto`：模型判断生成还是编辑。
- `generate`：强制新生成。
- `edit`：强制编辑上下文图片。

MVP 不依赖 Responses API。等文生图、图生图、遮罩稳定后再接入多轮。

## 前后端校验分工

前端负责：

- 即时校验参数。
- 隐藏无效参数。
- 展示成本和风险提示。

后端负责：

- 重复校验所有参数。
- 删除不支持的字段。
- 限制最大体积、最大 `n`、最大尺寸。
- 防止用户绕过前端直接请求高成本参数。
